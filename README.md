# Project Handoff Summary — Sheet Ingestor (Electron + Python + React)
# Version 2 — updated after UI/debug session

---

## Goal
Desktop app (Windows .exe) that ingests 2 Excel files, lets user map sheet names per file via a table UI with preview, then passes the mapping to a validation stage (not yet specified by user — to be defined in a future session).

---

## Stack (confirmed)
- **Frontend**: Electron (shell/window/file-dialogs) + React 18 + Vite (renderer UI). User explicitly chose React+Vite over vanilla JS mid-project — vanilla version was built first, then replaced.
- **Backend**: Python Flask, run as a local REST API on `127.0.0.1:5123`. Confirmed mandatory — without it Electron cannot call Python conversion/sheet-reading logic, packaged or not.
- **Comms**: Electron main process spawns Python as a child process on app start (`spawn`), then talks to it over HTTP (axios) via Flask endpoints. Renderer talks to main process via `contextBridge`/IPC (`preload.js`), never touches Node/Python directly (`contextIsolation: true`, `nodeIntegration: false`).
- **Packaging**: PyInstaller bundles `app.py` → `app.exe` (must be built first, manually, before electron-builder run). electron-builder (`npm run dist:win`, NSIS target) packages Electron + React build + the PyInstaller exe (via `extraResources`) into final Windows installer.

---

## Confirmed business logic
1. User selects 2 files via OS file picker (filter: `.xls`, `.xlsx`, `.xlsm`).
2. Files are converted to `.xlsx` via Python (`converter.py`, uses pandas + openpyxl, xlrd engine for legacy `.xls`).
3. Files are identified **by filename substring**, not fixed names — sheet names inside are NOT fixed either:
   - Filename contains "plan operations" → multi-sheet file → need **2** sheets picked from it.
   - Filename contains "net trust assets" → single-sheet file → need **1** sheet picked from it.
4. After conversion, mapping screen shows a table, exactly 3 rows (confirmed layout by user):

   | File Name | Sheet | Preview |
   |---|---|---|
   | plan operations file | dropdown (sheet 1) | Preview btn |
   | plan operations file | dropdown (sheet 2) | Preview btn |
   | net trust assets file | dropdown (sheet) | Preview btn |

   Filename **repeats** per sheet-slot row (not merged/grouped cells). Each row has independent dropdown (populated from that file's actual sheet names via openpyxl) and its own Preview button.
5. Preview button opens a modal showing the first ~20 rows of the selected sheet (Python reads via openpyxl, returns JSON columns+rows, rendered as HTML table in modal).
6. "Confirm" button currently just logs/alerts the final `{filePath, selectedSheet}` array for the 3 rows — **this is the handoff point to the validation stage, which the user has NOT specified yet**. Next session should ask user for validation rules before building further.

## Bugs fixed this session

### 1. `window is not defined` crash on startup
- **Cause**: A debug `console.log(window.api)` line was accidentally placed in `main.js` (Node process). `window` doesn't exist in Node.
- **Fix**: Removed that line from `main.js`. `console.log(window.api)` is only valid in Electron DevTools console (browser context).

### 2. App loading wrong `index.html` (old vanilla build)
- **Cause**: Stale `dist-renderer/` folder contained the old vanilla HTML with `<script src="/upload.js">` and title "Ingest Files". Electron was loading this instead of the Vite dev server.
- **Fix**: Deleted `dist-renderer/` folder. In dev mode (`npm run dev`), Electron always loads `http://localhost:5173` via `ELECTRON_DEV=1` env var.

### 3. Topbar disappearing on MappingScreen
- **Cause**: Topbar was rendered inside `UploadScreen` only. When screen switched to `MappingScreen`, topbar was gone.
- **Fix**: Moved topbar into `App.jsx` so it renders above both screens persistently. Removed any topbar markup from `index.html` and individual screen components.

### 4. File picker only showing `.xlsx`
- **Root cause**: Windows file picker behavior — defaults to first filter. Added `{ name: 'All Files', extensions: ['*'] }` as second filter option in `main.js` `select-files` handler so user can fallback to all files if needed.

## Current `App.jsx` (source of truth)
```jsx
import React, { useState } from 'react';
import UploadScreen from './components/UploadScreen.jsx';
import MappingScreen from './components/MappingScreen.jsx';

export default function App() {
  const [screen, setScreen] = useState('upload');
  const [ingestPayload, setIngestPayload] = useState(null);

  function goToMapping(payload) {
    setIngestPayload(payload);
    setScreen('mapping');
  }

  function goBackToUpload() {
    setScreen('upload');
    setIngestPayload(null);
  }

  return (
    <>
      <div className="topbar">
        <span className="topbar-logo">Deloitte.</span>
        <div className="topbar-divider"></div>
        <span className="topbar-app">Sheet Ingestor</span>
      </div>
      {screen === 'mapping' && ingestPayload
        ? <MappingScreen payload={ingestPayload} onBack={goBackToUpload} />
        : <UploadScreen onContinue={goToMapping} />
      }
    </>
  );
}
```

## Important gotchas for next session
- Topbar is in `App.jsx` only — do NOT add it inside any screen component.

- Old vanilla files (`renderer/index.html`, `renderer/mapping.html`) are obsolete — if they exist locally, do not mix them in.
